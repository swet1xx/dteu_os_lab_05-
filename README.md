## Контрольні питання

1. **Що таке гілка в Git?**
   Гілка в Git - це легкий переміщуваний вказівник на один з комітів. Вона дозволяє розробникам відгалужуватися від основної лінії розробки (наприклад, main) та продовжувати роботу самостійно, не впливаючи на основну кодобаз.

2. **Яка різниця між `git merge` та `git rebase`?**
   `git merge` створює новий коміт-злиття, який об'єднує історії двох гілок. `git rebase` переміщує послідовність комітів однієї гілки в основу іншої, створюючи лінійну історію. Rebase робить історію чистішою, але змінює історію комітів, що небезпечно для вже опублікованих змін.
